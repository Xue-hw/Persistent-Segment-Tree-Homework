# 数据结构课程设计：可持久化线段树（主席树）的实现与应用

## 📌 项目简介

本项目为数据结构课程设计作业。主要解决的问题是：**给定一个静态数组，要求利用可持久化线段树（主席树），在保证时间与空间效率的前提下，支持对任意区间 $[l, r]$ 进行多次“第 $k$ 小元素”的快速查询。** 

在此基础上，本项目还拓展了**元素排名查询**、**区间求和**，以及作为加分项的**区间不同元素个数查询**。并提供了一套直观的**终端可视化侧向打印工具**，方便观察持久化机制中节点的历史状态回溯与内存复用情况。

---

## 🚀 功能特性

1. **数组离散化处理**
   - 面对值域跨度大或包含负数的数组，先进行去重并排序，建立原始值与离散化下标的映射。
   - 大幅减小了线段树的深度，提高了查询效率，降低了空间开销。
2. **构建可持久化权值线段树**
   - 插入新元素时，仅仅复制从根到叶子的变更路径（产生 $\log_2 N$ 个新节点），绝大部分节点与上一历史版本**共享**。
   - 记录每次插入操作后生成的新根节点，形成版本控制。
3. **多维度的区间查询**
   - **核心：区间第 $k$ 小查询** —— 利用版本 $R$ 与 版本 $L-1$ 的树进行“节点相减”（前缀和思想），获得目标区间内的值域分布状态，从而向下二分搜寻第 $k$ 小。
   - **加分项：区间不同元素个数查询** —— 额外维护了一棵**基于数组下标建树**的可持久化线段树，通过追踪元素的“最后出现位置（last_pos）”来动态抹除重复贡献，完成独立的区间去重计数。
4. **可视化树结构展示**
   - 采用右-根-左（带缩进）的侧向打印逻辑，在终端中直观输出树的形态。
   - 清晰展示 `count` (子树元素总个数) 和 `sum` (子树元素真实值累加和)，直观感受“历史共享”的过程。

---

## 🧠 难点与核心思想突破

- **理解可持久化思想与空间优化**：
  若每次更新都深拷贝整棵线段树，空间复杂度为 $O(N^2)$，显然不可接受。本项目实现了“只复制变更路径”的思想，使得单次修改的空间增量降至 $O(\log N)$。
- **离散化 + 权值线段树的双剑合璧**：
  将常规的“按位置划分”转变为“按数值大小排名划分”。通过双指针在两棵历史版本树中同步下潜，将树上差值视作区间内的实际频次。
- **多维度冲突与架构解耦 (双树架构)**：
  “第 $k$ 小”需要基于**值域（权值）**建树，而“区间不同元素”需要根据**数组位置（下标）**建树抹除旧状态。这两者存在维度上的根本冲突。本项目的健壮性体现在将这两种逻辑剥离开来，分别封装为 `PersistentSegmentTree` 与 `PersistentTreeDistinct`，职责清晰。

---

## 📊 复杂度分析 (N = 数组长度，M = 查询次数)

| 操作 / 指标 | 时间复杂度 | 空间复杂度 | 备注 |
| :--- | :--- | :--- | :--- |
| **离散化** | $O(N \log N)$ | $O(N)$ | 排序和哈希表构建 |
| **建树与插入** | $O(N \log N)$ | $O(N \log N)$ | 每次插入新建 $\approx \log_2 N$ 个节点 |
| **区间第 $k$ 小** | $O(\log N)$ | $O(1)$ | 同步向下遍历 $\log_2 N$ 层 |
| **区间不同元素** | $O(\log N)$ | $O(1)$ | 查单棵版本树的区间总和 |
| **整体查询 (M次)**| $O(M \log N)$ | - | 显著优于暴力法的 $O(M \cdot N \log N)$ |

## 🛠️ 项目结构与运行指南

### 文件组织
- `solution.py`: 核心算法实现，包含两个类的定义与逻辑。
- `test.py`: 自动化测试脚本，集成各种边界测试、正常查询测试，以及可视化过程打印。

### 运行环境
- 环境：Python 3.6+ 
- 依赖：仅使用标准库（未引入三方库，便于纯净展示数据结构底层实现）

### 执行命令
```bash
# 在终端中运行自动测试与展示代码
python test.py
```

### 预期输出展示（部分）
程序会自动运行两套流程：
1. **基础指令测试**：打印从数组中查询指定区间的第 $k$ 小、不同元素个数、以及不同特定版本的数值查询测试对比验证。
2. **可视化树状形态测试**：
```text
[可视化树结构 - 版本 2]
格式: [值域区间] (计数, 累加和)
     ┌── Leaf [4] (c:1, s:4)
 └── Node [1~4] (c:2, s:5)
 |   └── Node [1~3] (c:1, s:1)
 |   |   └── Leaf [1] (c:1, s:1)
```
*在每次插入元素后，你都能直观看到哪些节点被复用（地址未变），哪些路径上的节点 `count` 被增加了。*
